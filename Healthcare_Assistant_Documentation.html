1. Project Overview

Healthcare Assistant is an AI-powered medical diagnostic platform that analyzes medical images and connects patients with specialized doctors for appointments.

🎯 Primary Objectives

  • Provide real-time AI analysis of medical images (X-rays and retina images)
  • Match patients with appropriate doctors based on diagnosis
  • Facilitate secure appointment booking with scanning report upload
  • Enable doctors to review patient data and schedule appointments
  • Track appointment status and provide notifications

✨ Key Features

✓ Dual-modality AI analysis (X-ray & Retina)
✓ Real-time disease prediction
✓ Doctor recommendation system
✓ Appointment booking system
✓ Secure file upload
✓ Real-time notifications
✓ Patient & Doctor dashboards
✓ Firebase authentication

4. Patient Workflow (Complete Journey)

Step 1️⃣ : Login Page

URL: /login

Process:

  1. Select role: Patient or Doctor
  2. Enter email and password OR click Google Sign-In
  3. System validates against Firebase Auth
  4. Checks user role in Firebase Database
  5. Redirects: Patient → Home, Doctor → Dashboard

Step 2️⃣ : Home Page

URL: /

  • Hero section with project overview
  • Feature highlights and statistics
  • Call-to-action buttons: "Upload & Predict" and "View Results"
  • Sample images and disease categories

Step 3️⃣ : Choose Analysis Pathway

URL: /upload

Patient selects medical imaging type:

  • Chest X-ray Analysis → /upload/chest (Pulmonology)
  • Eye Disease Analysis → /upload/eye (Ophthalmology)

Step 4️⃣ : Upload Medical Image

Chest X-ray Diseases Detected:

  • ✅ Normal - No significant thoracic abnormality
  • 🫁 Pneumonia - Infectious lung opacity pattern
  • 🫀 Tuberculosis - TB-related pulmonary changes

Retina Diseases Detected:

  • ✅ Normal Retina - Healthy retinal structure
  • 👁️ Age-Related Macular Degeneration (AMD)
  • ☀️ Cataract - Lens opacity patterns
  • 🩸 Diabetic Retinopathy - Vascular lesions
  • 👓 Myopia - Myopic retinal changes
  • 🌊 Retinal Vein Occlusion (RVO)

Upload Specifications:

  • Format: PNG, JPG, JPEG, TIFF
  • Max Size: 25MB
  • Shows preview before submission

Step 5️⃣ : AI Analysis Processing

Backend Process:

  1. Receive image from frontend
  2. Resize image to 224x224 pixels
  3. Normalize pixel values
  4. Convert to tensor
  5. Load appropriate TensorFlow model
  6. Run inference (prediction)
  7. Generate GradCAM visualization (heatmap)
  8. Return: prediction, confidence, top_predictions, gradcam

Step 6️⃣ : View Results

URL: /result

Results displayed include:

  • Predicted disease name
  • Confidence percentage (visual bar)
  • Medical modality (X-ray or Retina)
  • Top predictions list with probabilities
  • Uploaded image preview
  • GradCAM visualization (AI attention heatmap)
  • Action buttons: Generate Report, Find Doctor, Clear Results

Step 7️⃣ : Generate Scanning Report

URL: /scan-report

Scanning report contains:

  • Patient information
  • Disease prediction and confidence
  • Top predictions list
  • Uploaded medical image
  • GradCAM heatmap visualization
  • Detailed medical analysis text

Storage:

  • localStorage (browser persistence)
  • sessionStorage (current session)
  • Firebase Storage (cloud backup)

Step 8️⃣ : Find Doctor

URL: /doctor-recommendation

Doctor Matching Process:

  1. Load all registered doctors from Firebase
  2. RAG engine analyzes disease prediction
  3. Match doctors by specialization
  4. Score doctors by:
  • Specialization match
  • Years of experience
  • Hospital location relevance
  • Overall match score

Doctors ranked and displayed in cards with:

  • Name, specialization, clinic
  • Location and experience
  • Availability and bio
  • "Book Appointment" button

Step 9️⃣ : Book Appointment Modal

Form Fields:

  • Patient Name (pre-filled from profile)
  • Age (required)
  • Gender - Male/Female/Other
  • Contact Number (required)
  • Disease/Prediction Result (pre-filled)
  • ⭐ Scanning Report Upload (MANDATORY)

Upload Features:

  • File size limit: 5MB (enforced)
  • Auto-retry: Up to 2 attempts on failure
  • Progress tracking: Shows upload percentage
  • Supported formats: PDF, DOC, DOCX, JPG, JPEG, PNG
  • Clear error messages for failures

Step 1️⃣0️⃣: Submit Appointment

Submission Process:

  1. Upload scanning report to Firebase Storage
    Path: scanning-reports/{patientUid}/{appointmentId}/{filename}
  2. Get download URL from Firebase
  3. Save appointment to Realtime Database
    Path: appointments/{appointmentId}
  4. Show success message

Data Saved:

  • Doctor info: ID, name, specialization, hospital
  • Patient info: Multiple UID variants for compatibility
  • Medical data: Disease, modality, report URL
  • Status: "pending"
  • Timestamps: requestTime, createdAt

Step 1️⃣1️⃣: Track Appointments

URL: /my-appointments

Real-time appointment dashboard displaying:

Step 1️⃣3️⃣: Review Patient & Report

When clicking "Accept & Schedule":

  1. Opens review page with appointment details
  2. Displays full patient information
  3. Shows disease diagnosis and prediction
  4. ⭐ Provides clickable scanning report link

Step 1️⃣4️⃣: Accept & Schedule Appointment

URL: /doctor-dashboard/appointment/{appointmentId}

Doctor Inputs:

  • Select checkup date (date picker)
  • Select checkup time (time picker)
  • Enter optional message (textarea)

Auto-Message Logic:

If doctor doesn't enter a message, system generates:

"Doctor accepted your request. Checkup is scheduled on [formatted date and time]."

Step 1️⃣5️⃣: Submit Acceptance & Send Notification

Firebase Updates:

Field Old Value New Value
status "pending" "accepted"
checkupDateTime None ISO date string
doctorMessage None Message from doctor
patientNotification None Same as doctorMessage
patientNotificationSeen - false (NEW badge)
acceptedAt - Current timestamp
respondedAt - Current timestamp

Result:

  • Appointment status changes to "accepted"
  • Checkup date/time saved to database
  • Doctor message sent to patient
  • NEW notification badge triggers for patient
  • Doctor redirected back to dashboard

Reject Appointment (Alternative)

Doctor can reject by clicking [Reject] button on dashboard:

  • status → "rejected"
  • patientNotification → "Your appointment request was rejected by doctor."
  • patientNotificationSeen → false
  • Patient sees rejection notification

7. Backend API & ML Models

Flask Prediction Server

Base URL: http://localhost:5000

Port: 5000 (default Flask port)

CORS: Enabled for frontend requests

API Endpoint: POST /predict

Request Format

POST /predict HTTP/1.1
Host: localhost:5000
Content-Type: multipart/form-data

Body:
  image: [binary file data]
  modality: "xray" or "retina"

Response Format

{
  "prediction": "Pneumonia",
  "confidence_percent": 87.5,
  "top_predictions": [
    {
      "label": "Pneumonia",
      "confidence": 87.5
    },
    {
      "label": "Tuberculosis",
      "confidence": 8.2
    },
    {
      "label": "Normal",
      "confidence": 4.3
    }
  ],
  "gradcam": "base64_encoded_heatmap_image",
  "modality": "xray"
}

ML Models

Modality Model File Format Diseases
Chest X-ray medical_model.keras TensorFlow Keras Normal, Pneumonia, Tuberculosis
Retina Fundus EfficientNetB0_ODIR_OfflineAug.h5 HDF5 Format Normal, AMD, Cataract, DR, Myopia, RVO

Model Processing Pipeline

  1. Receive medical image from frontend
  2. Image validation and preprocessing
    • Resize to 224x224 pixels
    • Normalize pixel values
    • Convert to tensor format
  3. Load appropriate TensorFlow Keras model based on modality
  4. Run inference (forward pass through neural network)
  5. Extract prediction results
    • Top class prediction
    • Confidence score (0-100%)
    • Probabilities for all classes
  6. Generate GradCAM visualization (attention heatmap)
  7. Return JSON response to frontend

RAG Engine (Doctor Matching)

Advanced doctor recommendation using:

  • Chroma: Vector database for embeddings
  • LangChain: Vector search and matching
  • Embeddings: Convert doctor profiles to vectors
  • Similarity Search: Find best matching doctors
  • Specialization Scoring: Weight by specialty relevance

Data Matching Strategy

Write-side: Store all identity variants

  • Patient IDs: patientId, patientUid, userId, uid
  • Emails: patientEmail, email
  • Names: patientName, name

Read-side: Match on ANY variant

  • Ensures appointments always found
  • Compatible with legacy records
  • Handles incomplete field scenarios

11. Troubleshooting Guide

Issue Cause Solution
"Cannot reach prediction server" Backend not running Start Flask server on port 5000
"Firebase permission denied" Database rules not configured Update Firebase Database rules
"Model not found" Models not in backend/model/ Place model files in correct directory
"Upload timeout" Network or file size issue Check connection; ensure file ≤ 5MB
"Doctor not found" Incomplete doctor profile Complete doctor signup details
"Appointment not saving" Database connection issue Check Firebase credentials
"GradCAM not displaying" Base64 encoding issue Verify image preprocessing
"Login fails repeatedly" Role mismatch or auth error Check user role in database
